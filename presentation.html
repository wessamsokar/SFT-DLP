SFT-DLP

Secure File Transfer & Data Leakage Prevention

A Comprehensive Cybersecurity Solution

Created for Cryptography Project

1. Introduction

SFT-DLP is a local-first desktop application designed to secure sensitive file transfers.

  • Strong Encryption (AES-256-GCM)
  • Automated Policy Enforcement (DLP)
  • Auditable Sharing Workflows

2. The Problem

Data Leaks & Insecure Sharing

  • Employees accidentally sharing sensitive files (credit cards, IDs).
  • Unencrypted data transfers leading to interception.
  • No trace or audit of who shared what, and when.

3. The Solution

SFT-DLP combines three pillars of security:

  1. Cryptography: Bulletproof encryption.
  2. Policy Controls: Real-time content scanning.
  3. Monitoring: Immutable audit logs.

4. Encryption Engine

Ensuring data confidentiality and integrity.

  • Algorithm: AES-256-GCM (Authenticated Encryption)
  • Key Derivation: PBKDF2-HMAC-SHA256
  • Iterations: 600,000 (Protection against brute-force)
  • Protects against tampering during transfer.

5. DLP Engine

The core firewall for your files.

  • Scans files before they are allowed to be shared.
  • Uses regular expressions to find sensitive IDs, Credit Cards.
  • Blocks unauthorized file extensions entirely.

6. Secure Local Sharing

How files move safely.

  • Generates a unique, secure access token.
  • Enforces expiration times on all shared links.
  • Recipient must present token before the time limit expires.

7. Audit & Monitoring

Complete visibility into system usage.

  • Logs every encryption, decryption, and sharing attempt.
  • Records DLP interventions (Blocks & Warnings).
  • Stored securely in a local SQLite database.
  • Real-time dashboard for administrators.

8. System Workflow

A typical secure transfer process:

  1. User selects a file to share.
  2. DLP Engine scans the file contents.
  3. If safe, Encryption Engine secures it.
  4. A secure token is generated and given to recipient.
  5. Action is recorded in the Audit Logs.

9. Technical Security Model

  • Path Traversal Protection: Prevents unauthorized directory access.
  • Unique Nonces: AES-GCM uses a random nonce per file to prevent replay attacks.
  • Authentication Tags: Verifies file hasn't been altered.

10. UI & Architecture

Built with Python & PyQt5

  • Modern, responsive Cyber-Security Dashboard interface.
  • Modular structure (Core Engines, Database layer, GUI layer).
  • Local-first design ensures no data leaves the user's control.

11. Conclusion

Mission Accomplished

SFT-DLP successfully demonstrates how to integrate strong cryptography with practical business rules (DLP) to create a robust, auditable file transfer system.
